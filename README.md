# SteamArt

Подтягивает арты (**обложки**, баннеры, hero, logo, icon) для **non-Steam игр** из
[SteamGridDB](https://www.steamgriddb.com) и кладёт их в Steam.

Зачем: при добавлении игры в Steam как non-Steam (например, через **Hydra Launcher**) часто не
подтягивается «Основное изображение» — вертикальная обложка 600×900, и в библиотеке остаётся
серая заглушка. SteamArt дозаливает недостающее, не трогая то, что уже есть.

## Возможности

- 🖼️ **Визуальный выбор артов** — сетка вариантов с SteamGridDB по 5 типам (обложка, баннер,
  hero, logo, icon), клик → установить.
- ⚡ **Авто-дозаливка** — одним нажатием заполняет все недостающие арты по всем аккаунтам.
- 🔎 **Ручной поиск** игры, если авто-сопоставление промахнулось.
- 🧹 **Очистка осиротевших** артов (игр, которых больше нет в Steam).
- Современный интерфейс (стекло + неон) в нативном окне (pywebview / Edge WebView2).
- Есть и CLI-режим без GUI.

## Установка и запуск

Нужен **Python 3.x** (Windows) и Steam.

1. Получи бесплатный API-ключ: [steamgriddb.com](https://www.steamgriddb.com) → Preferences →
   API → Generate. Вставь его в окне (кнопка «Ключ») или положи в файл `steam_art.key` рядом
   со скриптом (одна строка). Можно также задать переменную окружения `STEAMGRIDDB_API_KEY`.
2. Запусти **`run_app.bat`** — он доустановит зависимости (Pillow, pywebview) и откроет окно.

После установки артов **перезапусти Steam**, чтобы он их подхватил.

### CLI (без интерфейса)

```bat
python steam_art.py                 :: дозалить недостающее по всем аккаунтам
python steam_art.py --dry-run       :: показать, ничего не качая
python steam_art.py --types cover   :: только обложки
python steam_art.py --clean         :: удалить осиротевшие арты
python steam_art.py --account <uid> :: один аккаунт
python steam_art.py --force         :: перезаписать существующее
```

## Как это устроено

- `steam_art.py` — движок (только stdlib) + CLI.
- `steam_art_app.py` — локальный HTTP-сервер + нативное окно (pywebview).
- `web/` — фронтенд (HTML/CSS/JS, без сборки).

Подробности и заметки для разработки — в [CLAUDE.md](CLAUDE.md).

## Безопасность

`steam_art.key` (твой API-ключ) в `.gitignore` и не попадает в репозиторий. SteamArt только
**читает** `shortcuts.vdf` и **пишет** картинки в `userdata\<uid>\config\grid\` — сами ярлыки
Steam не меняются.
